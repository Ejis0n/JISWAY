# SEO・流入について

## すでにやっていること（技術SEO）

| 項目 | 状態 |
|------|------|
| **sitemap.xml** | 全ページ（TOP、カテゴリ、商品約1900）を出力 |
| **robots.txt** | Allow /、Disallow /admin・/api、Sitemap 指定 |
| **canonical** | 各ページで canonical URL 指定 |
| **メタ情報** | title / description / Open Graph（商品・カテゴリ） |
| **構造化データ** | Product（価格・在庫・画像）、FAQ、BreadcrumbList |
| **Google Search Console** | サイト認証済み |
| **Google Analytics** | G-78DN3QC4VK 設置 |
| **IndexNow** | Bing 等への即時通知 API あり |
| **モバイル** | レスポンシブ・タッチ対応 |
| **HTTPS・HSTS** | 本番で有効 |

技術的な「抜け」はほぼありません。

### スマホ最適化（実施済み）

- **viewport** 指定（width=device-width, initial-scale=1）
- **タッチ操作**: ボタン・リンクは `min-h-[44px]` などでタップしやすいサイズ
- **レイアウト**: ヘッダーはモバイルで折りたたみメニュー、商品表は横スクロール（`overflow-x-auto`）で切れない
- **フォント・余白**: 読みやすいサイズと余白で狭い画面でも操作しやすい
- **CSP**: `img-src` で HTTPS 画像を許可しつつセキュアに表示

## 流入が少ない理由（SEOだけとは限らない）

1. **ニッチな商材**  
   「JIS ボルト・ナット・ワッシャー」「海外調達」は検索ボリューム自体が少なめです。SEOを整えても、元々の検索数が少ないと流入は伸びにくいです。

2. **サイトが新しい**  
   公開してから数ヶ月程度だと、Google がまだ全ページを評価しきっていないことがあります。Search Console の「カバレッジ」「サイトマップ」でインデックス数を確認し、問題があれば「URL 検査」で再クロール依頼するとよいです。

3. **被リンクが少ない**  
   他サイトからのリンクがほとんどないと、検索での「重要度」が上がりにくいです。技術SEOより、業界サイト・取引先・SNS などからのリンク獲得が効くことが多いです。

4. **キーワードとの一致**  
   「JIS bolt」「hex nut procurement」「日本 ボルト 輸出」など、想定している検索語で実際に検索されているか、Search Console の「検索パフォーマンス」で確認するとよいです。ここでクリックが少ない場合は、文言の見直しやコンテンツ追加の余地があります。

5. **ショッピングタブ・リッチリザルト**  
   Product 構造化データは入れてあるので、商品ページが増え・インデックスされれば、ショッピングタブやリッチリザルトに出る可能性はあります。反映には時間がかかることがあります。

## このあとやるとよいこと（優先度の目安）

### すぐできる

- **Search Console の確認**  
  - 「カバレッジ」でエラー・警告がないか  
  - 「サイトマップ」で送信した URL が取り込まれているか  
  - 「検索パフォーマンス」でどんなクエリで表示されているか  
- **主要キーワードの確認**  
  - 想定キーワードで Google 検索し、自サイトが何位付近にいるか、競合がどんなタイトル・説明かを見る。

### 中期的に

- **コンテンツの足し**（TOP に追加済み）
  - **What is JIS?**: TOP に JIS（日本工業規格）と JIS ファスナーの説明を掲載済み。スマホでも読みやすい余白・フォントで表示。
  - **How procurement works**: カート → 決済 or 見積承認 → 日本で調達 → 発送の 3 ステップを TOP に掲載済み。
  - カテゴリページにも「JIS Bolts とは」など 1〜2 文の説明を足すと有効な場合があります。
- **被リンク**
  - 業界ディレクトリ、取引先サイト、関連記事などからリンクをもらう。
  - 技術ブログ・調達事例・規格説明ページで JISWAY に言及してもらう。
- **SNS・広告**
  - 検索以外（LinkedIn、業界メディア、Google 広告など）で認知を増やし、ブランド検索や直リンクを増やす。

### 運用で確認したいこと

- **Google アナリティクス**  
  - どのページがよく見られているか、どの流入経路（Organic / Direct / Referral）が多いか。  
- **Search Console**  
  - 表示回数は増えているか、クリックされているクエリは何か。  
- **商品ページのインデックス**  
  - 重要な商品 URL を「URL 検査」で送信し、インデックス・リクエストしておく。

## あとできること（サイト改善の候補）

やると効果がある・負荷が小さいものから並べています。必要に応じてピックアップしてください。

| 内容 | 効果・備考 |
|------|------------|
| **フッターに Support / Make an offer リンク** | 問い合わせ・オファーへの導線が増える。 |
| **スキップリンク（Skip to main）** | キーボード・スクリーンリーダー利用者向け。アクセシビリティ向上。 |
| **カテゴリページに 1〜2 文の説明** | 「JIS Bolts」など各カテゴリの短い説明で、検索・理解しやすく。 |
| **GA のコンバージョン目標** | 購入完了・見積送信・オファー送信を目標にすると、施策の効果が見やすい。 |
| **Cookie バナー（EU 向け）** | 欧州からのアクセスがある場合は、同意バナーを検討。 |
| **エラー監視（Sentry 等）** | 本番の JS エラーを記録すると、不具合に気づきやすい。 |
| **管理画面：新着オファー時のメール** | 客からオファーが来たら管理者にメール通知（既存の sendEmail で可能）。 |
| **配送目安の表示** | 商品 or カートに「目安 2〜3 週間」などがあると安心感が出る。 |
| **FAQ ページの拡充** | よくある質問をまとめた専用ページがあると、検索・離脱防止に有効。 |
| **多言語・多通貨** | 日本語ページや JPY 表示など、必要になったら検討。 |

実装コストが小さいものは「フッターリンク」「スキップリンク」「カテゴリ説明」です。

## Search Console のインデックス状況（補足）

- **サイトマップ**: `/sitemap.xml` 送信済み・成功、約 1,930 URL 検出。
- **登録済み**: 約 398 ページ。**未登録**: 約 1,600 ページ。
- **未登録の主な理由**:
  - **重複（正規未選択）**: `/quote?sku=xxx` など 2 件 → **対応済み**: `/quote` に `canonical="/quote"` を設定し、正規 URL を明示。
  - **検出・インデックス未登録**: 1,435 件。Google がクロール優先度でまだインデックスしていない状態。時間経過や「URL 検査」でのインデックス登録リクエストで改善する場合あり。
  - **クロール済み・インデックス未登録**: 163 件。クロールはされているが、品質・重複等の判断で未採用。重要な URL は「URL 検査」から登録をリクエスト可能。
- **検索パフォーマンス**: 3 か月でクリック約 11・表示約 82 など、ニッチなため数値は少なめ。日本からのクリックが最多。

## まとめ

- **技術SEO**は、sitemap・robots・canonical・メタ・構造化データ・GSC・GA など、必要な要素はそろっています。  
- **流入が少ない主因**は、SEOの不備というより、  
  - ニッチな商材で検索数がもともと少ない  
  - サイトが新しく・被リンクが少ない  
  - 狙うキーワードとコンテンツの一致がまだ弱い  
  といった要因の可能性が高いです。  
- このあとは、**Search Console と Analytics の数字を見ながら**、キーワード・コンテンツ・被リンク・認知施策を少しずつ足していくのがおすすめです。
